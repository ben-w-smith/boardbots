# Research Documentation

This directory contains research findings, experiments, and benchmarks.

## Purpose

- Document experiments and their outcomes
- Store benchmarks and performance data
- Keep notes on third-party libraries or patterns investigated
- Preserve knowledge from one-off investigations

## Types of Research

1. **Performance**: Benchmarks, optimization attempts, profiling results
2. **Architecture**: Design explorations, alternative approaches considered
3. **Integration**: Third-party library evaluations, API investigations
4. **Algorithm**: AI improvements, resolution algorithm alternatives

## File Naming

Use descriptive names that indicate the research topic:
- `ai-performance-benchmark.md`
- `websocket-reconnection-strategies.md`
- `hex-rendering-optimization.md`

## Guidelines

1. **Include raw data**: Don't just summarize - include actual measurements
2. **Note methodology**: How were measurements taken?
3. **Document environment**: What hardware, Node version, etc.?
4. **Be honest**: Document failed approaches too
5. **Update status**: Mark research as complete or abandoned
